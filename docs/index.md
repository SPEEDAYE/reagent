# docs/

Repository documentation is organized by code-facing purpose.

| Path | Purpose |
|---|---|
| `feature/` | Feature notes and user-facing behavior changes. |
| `exec-doc/` | Execution plans and change records. |
| `design-doc/` | Module design notes and architecture details. |
| `references/` | API, deployment, and integration reference material. |
| `system-visualization/` | Pipeline, code function, LLM call, prompt, and external-call maps. |

Writing materials such as LaTeX, BibTeX, review replies, and submission notes do not belong in this repository.
