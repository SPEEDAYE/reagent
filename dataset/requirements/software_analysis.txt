我们计划开发一个自动化软件源代码审查平台，该平台需要提供如下功能：
1. GitLab Webhook 集成
2. 自动化代码分析 (Automated Analysis)
  1. 自动生成 MR/PR 评论
  2. 安全漏洞检测 (Security Check)
  3. 代码风格与规范检查 (Linting)
  4. 性能优化建议
3. 多语言支持
4. 自定义审查规则 (Prompt Engineering)
5. 审查质量反馈机制
  1. 审查报告与总结
  2. 开发者接受率统计（一次评审 N 个问题。接受整改和拒绝整改 的比例）
  3. 代码质量趋势分析（AI 审查出来的问题数量变化率）
  4. 常见问题热力图
  5. 审查效率统计（AI 完成一次审查的时间）

这是我们对项目的时间限制、技术栈要求以及评审标准：
时间： 1-2 周
前端：react、后端 python
AI：GPT、Claude、Gemini
评审标准包括：
1. 功能的匹配度（没有过度精简，没有过度扩充）；
2. 功能的可用性及准确性；
3. 可用性和流畅度；
4. 架构的性能和高可用的考虑；
