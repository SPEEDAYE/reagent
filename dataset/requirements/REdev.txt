我们计划研发一个面向开发者的面向需求开发的IDE拓展，以 VS Code for the Web 作为基础编辑与协作环境，通过引入 AI Agent 驱动的需求工程能力，支持用户在统一的平台中完成需求获取、需求分析、需求规约化与需求验证等核心需求工程活动。
在功能层面，平台应支持用户在 Web IDE 中开展AI Agent的需求工程活动，包括但不限于：
需求获取、需求分析、需求规约化、需求验证
在系统设计上，平台应在 VS Code Web 的基础能力之上扩展面向需求工程的交互能力，至少包括：
在 VS Code for the Web 提供的文本编辑与文件管理能力基础上，系统应支持用户在同一 Workspace 中创建、编辑和组织需求相关文档，使需求工程活动能够以项目级制品的形式持续演进，而非零散的对话或临时文本。
在 VS Code Web 支持自定义 Panel 与 View 的能力基础上，系统应提供专用于需求工程的 Agent Panel，用于承载不同需求工程阶段的智能分析过程，并使需求分析、规约与验证结果能够与当前编辑的需求文档保持上下文关联。
在 VS Code Workspace 与文件系统抽象的基础上，系统应支持将需求分析和规约结果以明确的需求制品形式进行管理，并允许用户在 Artifact Panel 中查看、修改和复用这些需求制品，以支撑需求的版本演进与可追溯性。

（先去掉，后面加上）在 VS Code 内置差异对比（Diff）能力的基础上，系统应支持对需求在不同迭代阶段的版本进行对比，以帮助用户理解需求调整带来的变化及其影响。
